Fundamentos

Trilha 0

Pré-requisitos ensinados sob demanda — cada um entra na primeira vez que aparece nas outras trilhas, do básico ao avançado.

  1. Markdown Títulos e listas → frontmatter, tabelas, a "língua nativa" dos LLMs
  2. Terminal, shell e CLI O balcão, o atendente e o cardápio: comandos, caminhos, e por que o Claude Code é um CLI
  3. Formatos de dados JSON, YAML e como se descreve uma ferramenta pra um LLM
  4. Git O "histórico de versões" do mundo do código
  5. APIs e HTTP Como programas conversam entre si pela internet
  6. Arquivos e sistema Pastas, arquivos ocultos, extensões e executáveis

Claude Code na prática

Trilha A

O laboratório: cada conceito vira algo que você faz de verdade na sua máquina.

  1. Fundamentos do Claude Code O ciclo do agente, permissões, primeiros comandos — veja a aula de exemplo
  2. Contexto e memória A "mesa de trabalho" do LLM; CLAUDE.md; memória entre sessões
  3. Ferramentas nativas As "mãos": ler, escrever, buscar, executar
  4. Skills e comandos Transformar o agente em especialista (como esta skill!)
  5. Hooks e configurações Regras automáticas e permissões sob medida
  6. Subagentes O mestre de obras e seus ajudantes
  7. MCP Plugar o agente em outros serviços (Figma, Slack, Gmail…)
  8. Workflows e orquestração Vários agentes trabalhando em conjunto
  9. Git + Claude Code Versionar, revisar e colaborar como os profissionais
  10. Automação Agentes que trabalham sozinhos, em horários agendados

Arquitetura de IA

Trilha B

A teoria por trás: entender o que acontece dentro da máquina — sem matemática pesada.

  1. Como um LLM funciona O autocompletar gigante: tokens, previsão, temperatura
  2. Anatomia de um agente Cérebro + mãos + objetivo; o ciclo pensa→age→observa
  3. Engenharia de contexto System prompts e a arte de instruir um LLM
  4. RAG Como dar uma "biblioteca" pesquisável ao modelo
  5. Design de ferramentas Como desenhar boas "mãos" — puro design, aliás
  6. Arquiteturas multiagente Orquestradores, painéis de juízes, verificação cruzada
  7. Avaliação (evals) Como saber se a IA está boa — "parece bom" não basta
  8. Segurança e confiabilidade Prompt injection, permissões, o humano no circuito
  9. Custo e latência Escolher o modelo certo pra cada tarefa
  10. Projeto final Ver abaixo 👇

🏁 Projeto final: o seu agente

Ao fim do curso, você desenha e constrói um agente completo, escolhido conforme a sua profissão — um designer pode criar um agente que revisa os textos de um projeto conforme o guide da marca; um analista, um agente que audita planilhas; um advogado, um revisor de contratos. O professor adapta o projeto a você.

As trilhas são intercaladas (uma aula prática, uma teórica), os fundamentos entram quando o assunto pede, e a ordem se adapta à curiosidade de cada aluno. O progresso fica salvo entre sessões — o professor sempre lembra de onde vocês pararam.
