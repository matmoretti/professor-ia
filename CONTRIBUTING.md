# Contribuindo com o Professor IA

Que bom que você quer melhorar o curso! Ele nasceu de uma aula real e cresce a cada aluno.

## Formas de contribuir

- **Achou um erro?** (conceito impreciso, link quebrado, português torto) → abra uma issue de *Erro no curso*.
- **Tem uma analogia melhor** para algum conceito? → abra uma issue de *Proposta de melhoria*. Analogias são o coração da metodologia — boas analogias novas valem ouro.
- **Quer propor um módulo ou aula nova?** → abra uma issue de *Proposta de melhoria* descrevendo o tema e onde ele entraria nas trilhas.
- **Quer mandar a correção pronta?** → faça um fork, edite e abra um pull request.

## Princípios do curso (leia antes de propor mudanças)

1. **Feito para leigos**: nenhum termo aparece sem explicação. Se sua mudança introduz jargão, explique-o no lugar.
2. **Analogias canônicas**: o curso reutiliza as mesmas analogias do início ao fim (LLM = autocompletar gigante; agente = cérebro + mãos; terminal/shell/CLI = balcão/atendente/prato...). Propostas de *substituição* precisam ser melhores que a atual — não apenas diferentes.
3. **Mão na massa primeiro**: toda aula tem demo ao vivo e exercício do aluno. Aula só teórica não entra.
4. **Uma coisa de cada vez**: máximo 1 conceito central por aula.
5. **Idioma**: português brasileiro, tom informal mas tecnicamente preciso.

## Testando suas mudanças

Copie o conteúdo de `skill/` para `~/.claude/skills/professor-ia/` (ou `C:\Users\SEU_USUARIO\.claude\skills\professor-ia\` no Windows), abra o Claude Code e diga "bora estudar". O professor deve seguir sua versão da skill.
